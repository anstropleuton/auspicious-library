\doxysubsection{Detailed Description}
Command line argument parser. 

This supports both the new POSIX-\/style arguments and the old Microsoft-\/style arguments. Each argument from the command line is parsed and is also modified internally for interpretation.

New POSIX-\/style arguments\+:
\begin{DoxyItemize}
\item Long Argument\+: Each long argument starts with two hyphen ({\ttfamily -\/-\/}) and consists of one or more characters for recognition. Such as {\ttfamily -\/-\/long-\/argument}. The allowed characters are not an {\ttfamily =} sign or a {\ttfamily \+:} symbol. If used, the behavior is undefined.
\item Short Argument\+: Each short argument starts with one hyphen ({\ttfamily -\/}) and consists of one character for recognition. Such as {\ttfamily -\/a}. If multiple characters are found, such as {\ttfamily -\/abc}, it internally splits each character as its own separate short arguments, such as {\ttfamily -\/a}, {\ttfamily -\/b}, {\ttfamily -\/c}. The allowed characters are not an {\ttfamily =} sign or a {\ttfamily \+:} symbol. If used, the behavior is undefined.
\item Use of a {\ttfamily -\/-\/} with no additional characters indicates end of parsing and rest of the arguments are unparsed.
\end{DoxyItemize}

Old Microsoft-\/style arguments\+:
\begin{DoxyItemize}
\item There is no dedicated section for the old Microsoft-\/style arguments, often known as "{}switches"{}.
\item Instead, each switch is treated as a long option if the switch is more than one character for recognition, such as {\ttfamily /switch}.
\item If a switch is one character for recognition, such as {\ttfamily /s}, it is treated both as a long option and a short option (short name is matched first).
\item If a switch is not matched, you can optionally make it try to match case insensitively.
\end{DoxyItemize}

In both cases, arguments that contain an {\ttfamily =} sign or a {\ttfamily \+:} symbol, such as --option=value, -\/o\+:value, /switch=value or /s\+:value, are internally split from with the first found {\ttfamily =} sign or {\ttfamily \+:} symbol.

Each option or switch can have one or more value parameter. Each argument after an option or switch that do contain value parameter will be treated as the value to associate with an option or a switch. If an argument that is to be treated as a value looked like an option or a switch, it will not be treated as value associated to the previously appeared option or switch. In that case, if default argument is provided, it will use that. For example, let\textquotesingle{}s say you have parameter as such\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{password\_add\_option.parameters\ =\ \{}
\DoxyCodeLine{\ \ \ \ \textcolor{stringliteral}{"{}id"{}},\ \textcolor{stringliteral}{"{}password"{}},\ \textcolor{stringliteral}{"{}encryption-\/key"{}},\ \textcolor{stringliteral}{"{}require-\/password-\/to-\/decrypt"{}}}
\DoxyCodeLine{\};}

\end{DoxyCode}
 And you want to have default value for "{}encryption-\/key"{} and "{}require-\/password-\/to-\/decrypt"{} parameter. You would define as such\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{password\_add\_option.defaults\_from\_back\ =\ \{}
\DoxyCodeLine{\ \ \ \ \textcolor{stringliteral}{"{}DEFAULT\_ENCRYPTION\_KEY"{}},\ \textcolor{stringliteral}{"{}false"{}}}
\DoxyCodeLine{\};}

\end{DoxyCode}
 This will make ID and Password parameter as a mandatory parameter. \begin{DoxySeeAlso}{See also}
\doxylink{structauspicious__library_1_1ap_1_1option__template}{option\+\_\+template}.
\end{DoxySeeAlso}
You can use "{}..."{} as the last parameter if you want variadic parameters. i.\+e., all parameters not detected as option will be added to this vector. Use "{}parameter-\/name..."{} if you want to have at least one value.
